# Estamos por Venezuela — Brand Guide

Last updated: 2026-06-29. Audience: developers building on this codebase.

This document is the single source of truth for the visual design system. Read it before touching any UI file.

---

## TL;DR

Warm, human, editorial. Think Venezuelan earth tones and morning light — not cold tech grays. Every design decision should feel like it was made by people who care, not generated by an algorithm.

---

| Element | Value |

## Color Tokens

All tokens live under `brand.*` in `tailwind.config.ts`.

| Token | Hex | Tailwind class | Usage |
|-------|-----|----------------|-------|
| `canvas` | `#fdf4ea` | `bg-brand-canvas` | Main page background |
| `ink` | `#1a0d07` | `text-brand-ink` / `bg-brand-ink` | Primary text + dark sections/footer |
| `ember` | `#CF142B` | `bg-brand-ember` / `text-brand-ember` | Venezuelan red — emergency CTA, primary button |
| `sol` | `#E8A030` | `bg-brand-sol` / `text-brand-sol` | Warm amber-gold — donate strip, highlights |
| `cielo` | `#1E3E8F` | `bg-brand-cielo` / `text-brand-cielo` | Venezuelan blue — info badges |
| `tierra` | `#C4673A` | `text-brand-tierra` / `border-brand-tierra` | Terracotta — section dots, decorative accents |
| `warm-surface` | `#f9ece0` | `bg-brand-warm-surface` | Card tints, alternating section backgrounds |
| `warm-muted` | `#8c6a54` | `text-brand-warm-muted` | Secondary text on warm backgrounds |

### Rules
- Never use raw `slate-*` for backgrounds. Use `brand-canvas`, `brand-warm-surface`, or `brand-ink`.
- The three Venezuelan flag colors are `ember` (red), `sol` (gold), `cielo` (blue).
- `tierra` is the accent/decoration color — not a CTA color.

## Typography

### Fonts
- **Playfair Display** (`font-serif`, `--font-playfair`) — headings, hero titles, editorial numbers
- **Inter** (`font-sans`, `--font-inter`) — body text, UI labels, badges

Both are loaded via `next/font/google` in `app/layout.tsx`. Do not add more fonts.

### Scale

| Role | Size | Font | Notes |
|------|------|------|-------|
| Hero title | `5xl` → `8xl` | Serif | Responsive, leading-none |
| Section heading | `4xl` → `5xl` | Serif | |
| Card heading | `2xl` → `3xl` | Serif | |
| Body | `sm` → `base` | Sans | `leading-relaxed` |
| Section label | `11px` | Sans | Uppercase, `tracking-[0.2em]` |
| Badge | `xs` (10–11px) | Sans | Medium weight |

### Heading pattern (Hejl-inspired)
Use `<em>` inside Playfair headings to italicize one key word. Playfair's italic is highly distinctive.

```jsx
<h2 className="font-serif text-4xl">
  Todo en <em>un solo lugar</em>
</h2>
```

## Visual Motifs

### Section label
Every section opens with a label above the heading:

```jsx
<p className="section-label">Herramientas</p>
<h2 className="font-serif text-4xl">Todo en <em>un solo lugar</em></h2>
```

This renders as `● HERRAMIENTAS` in small terracotta-dotted caps.

### Background watermark
Large near-invisible serif word placed behind section content. Use sparingly — one per major content section.

```jsx
<span aria-hidden="true" className="absolute font-serif text-[22vw] text-brand-ink/[0.03] pointer-events-none select-none">
  Venezuela
</span>
```

### Warm gradient overlays
Replace all `from-slate-950/X` dark overlays with `from-brand-ink/X`. Same darkness, warmer undertone.

### Scrolling marquee
One instance on the home page only (between stats and urgent section). Uses `.marquee-track` class. Pauses on hover.

### Image treatment
- All image containers: `rounded-2xl overflow-hidden`
- No tilted images — inappropriate for crisis context

## Motion

| Effect | Status | Notes |
|--------|--------|-------|
| Fade-up on scroll | Keep | `InViewMotion` component, 0.7s ease |
| Scrolling marquee | Add | Home page only, CSS-only |
| Parallax | No | Inappropriate for users in distress |
| Hover rotations | No | Too playful |
| Ken Burns (hero) | Keep | 18s, subtle zoom |

## Page Structure Pattern

Every public page follows this rhythm:

1. **Header** — section label + serif heading + short description
2. **Content** — alternates between `canvas` and `warm-surface` section backgrounds
3. Navigational footer is `ink` (warm dark)

### Emergency pages (`/necesito-ayuda`)
- Use `ember` more prominently
- Keep `animate-pulse` on the primary CTA
- Dark hero acceptable here

---

## What Not to Do

- Don't use `slate-*` for backgrounds — use brand tokens
- Don't use `#FFD700` (electric yellow) — use `brand-sol` (`#E8A030`)
- Don't use `rounded-lg` on buttons — use `rounded-full` (pill)
- Don't use bare `gray-*` — everything goes through brand tokens
- Don't add new fonts
- Don't add emoji flags as decorative elements — the mark in the navbar is enough
